# ☁️ AWS Certifications Guide

Guia completo para entender, preparar-se e realizar as certificações da **Amazon Web Services (AWS)**.

Este documento explica:

* Estrutura das certificações
* Tipos de certificações
* Como se preparar
* Como agendar provas
* Como funcionam os exames
* Estratégias de estudo
* Custos e validade das certificações

---

# 📚 O que são certificações AWS

As certificações AWS validam o conhecimento de profissionais em **arquitetura, desenvolvimento, operações e especialidades na nuvem**.

Elas são reconhecidas globalmente e demonstram que o profissional sabe trabalhar com os serviços da AWS.

Benefícios:

* reconhecimento profissional
* melhores oportunidades de trabalho
* validação técnica de conhecimento
* acesso a comunidades e eventos AWS

---

# 🏗 Estrutura das certificações

As certificações AWS são organizadas em **4 níveis**:

| Nível        | Descrição                                  |
| ------------ | ------------------------------------------ |
| Foundational | Conhecimento básico de cloud               |
| Associate    | Conhecimento intermediário                 |
| Professional | Arquitetura avançada                       |
| Specialty    | Conhecimento profundo em áreas específicas |

---

# 🎓 Certificação Foundational

### ☁️ AWS Certified Cloud Practitioner

Certificação introdutória para quem está começando na nuvem.

### O que avalia

* conceitos de cloud computing
* serviços principais da AWS
* segurança
* faturamento e custos
* arquitetura básica

### Perfil recomendado

* iniciantes em cloud
* profissionais de negócios
* gerentes de projeto
* desenvolvedores iniciantes na AWS

### Informações da prova

| Item     | Valor                      |
| -------- | -------------------------- |
| Duração  | 90 minutos                 |
| Questões | 65                         |
| Tipo     | múltipla escolha           |
| Custo    | $100                       |
| Idiomas  | vários incluindo português |

# 📝 Como funcionam os exames

## Tipos de perguntas

Os exames incluem:

### Multiple choice

1 resposta correta entre 4 opções.

### Multiple response

Mais de uma resposta correta.

---

## Estilo das questões

A maioria das questões é baseada em **cenários reais**.

Exemplo:

> Uma aplicação precisa escalar automaticamente durante picos de acesso e reduzir custos quando o tráfego diminuir. Qual solução é mais adequada?

Você precisa escolher a **arquitetura correta**.

---

# 🧪 Sistema de pontuação

Pontuação varia de **100 a 1000 pontos**.

Nota mínima para aprovação:

**720 pontos**

---

# 🖥 Como fazer a prova

Você pode fazer o exame de duas formas.

## 1️⃣ Online (remoto)

Feito em casa com supervisão.

Requisitos:

* webcam
* microfone
* ambiente silencioso
* sem outras pessoas no local

A prova é monitorada por um **proctor online**.

---

## 2️⃣ Centro de testes

Prova presencial em centros autorizados.

Empresas parceiras:

* Pearson VUE
* PSI Services

---

# 📅 Como agendar o exame

1. Criar conta no portal da AWS Certification

[https://aws.amazon.com/certification](https://aws.amazon.com/certification)

2. Acessar o painel

3. Escolher a certificação

4. Escolher o provedor de exame

* Pearson VUE
  ou
* PSI Services

5. Escolher:

* data
* horário
* idioma

6. pagar o exame

# 📚 Como estudar para certificações AWS

Estratégia recomendada:

### 1️⃣ Curso estruturado

Plataformas populares:

* Pluralsight
* Udemy
* A Cloud Guru

---

### 2️⃣ Prática na AWS

Use o **Free Tier**.

Serviços essenciais para aprender:

* Amazon EC2
* Amazon S3
* AWS Lambda
* Amazon RDS
* Amazon DynamoDB

---

### 3️⃣ Simulados

Simulados são fundamentais porque:

* mostram o formato da prova
* ajudam a identificar pontos fracos
* aumentam a velocidade de resposta

---

# 🧠 Estratégias para passar na prova

Dicas importantes:

### 1️⃣ Aprenda quando usar cada serviço

Não basta saber **o que é**.

Você precisa saber **quando usar**.

---

### 2️⃣ Estude arquitetura

Conceitos importantes:

* alta disponibilidade
* fault tolerance
* escalabilidade
* elasticidade

---

### 3️⃣ Entenda custos

AWS sempre valoriza **soluções com menor custo operacional**.

---

### 4️⃣ Leia as perguntas com atenção

Frequentemente a questão pede:

* solução **mais barata**
* solução **mais escalável**
* solução **mais segura**
